プロジェクト概要

  • アプリ名
    • 英単語帳
  • どんなアプリか・作った目的
    • 英単語を入力し、生成AIを通して和訳・翻訳するアプリ
  • 開発期間
    • 約2か月間
  • 使用技術
    • Python
    • gemini api
      • 英単語の和訳の自然言語処理を担当
    • tkinter
      • アプリ全体のGUIを担当
    • sqlite3
      • ローカルで動作する簡易データベース

アプリを作った背景

  • なぜこのアプリを作ろうと思ったのか
    • 最新の論文（AIや機械学習の論文）を読むために英語の勉強を効率化するために自作しようと思い立ちました
  • どんな課題を解決したかったのか
    • 英単語を忘れた時に見返す時間の手間を効率よくしたいと思いました
  • ユーザーをどう想定したのか
    • どのボタンがどのような挙動をするのか視覚で判断できるように使いやすさを重視しました

アプリの機能一覧

  • ユーザーができること
    • 意味の知りたい英単語の入力
    • データベースに登録、削除、編集
    • 登録した英語を閲覧するテーブル表示
    • AIとの翻訳の評価・アドバイスをチャット形式で行える

工夫した点・難しかった点

  • 検索窓の作成を工夫しました。テーブルのウインドウが表示された際にデータベースからデータを取得するのでそれに合わせた検索機能を実装しないといけない欠点がありました。そこでウインドウが表示されている時には最新のデータをリストで保持し、それをコピーしたリストの中身を検索文字から文字列の前方一致で条件を付けてforで表示するように工夫しました。



開発中に得た学び

  • API設計で.envファイルを用いて不正アクセスと管理のしやすさを学びました
  • MVCの型を利用し、運用保守や機能追加をしやすいよう構築できるようになりました

今後追加したい機能

  • ログイン機能
  • 英検やTOEICの問題を出題する試験形式の機能
  • csv形式で出力する機能
